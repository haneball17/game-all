## 🏗 架构设计

### 模块结构

```
Payload/
├── modules/
│   ├── helper/
│   │   └── helper.cpp          # 现有Helper模块
│   ├── sync/
│   │   └── sync.cpp            # 现有Sync模块
│   └── stealth/                # 新增隐蔽模块
│       ├── peb_utils.h         # PEB操作工具
│       ├── peb_utils.cpp       # PEB操作实现
│       ├── module_hider.h      # 模块隐藏接口
│       └── module_hider.cpp    # Unlink实现
└── entry.cpp                   # Payload入口

新的模块职责:
├─ peb_utils: 封装PEB访问、链表遍历等底层操作
├─ module_hider: 实现Unlink逻辑，提供隐藏/显示接口
└─ 现有模块: 调用module_hider实现自我隐藏
```

### 设计原则

```
1. 最小侵入
   → 不修改现有Helper/Sync核心逻辑
   → 通过独立模块提供功能

2. 可配置开关
   → 支持运行时启用/禁用
   → 便于调试和问题排查

3. 平台兼容
   → 当前支持x86
   → 预留x64扩展能力

4. 安全性
   → 失败时不影响DLL正常运行
   → 完善的错误处理
```

## 💻 代码框架

> 完整代码实现请参考 [04-代码模板.md](./04-代码模板.md)

### 主要组件

#### 1. PEBUtils 类

负责PEB访问和模块遍历：

```cpp
namespace stealth {

class PEBUtils {
public:
    // 获取当前进程PEB
    static PVOID GetCurrentPEB();
    
    // 获取LDR数据
    static PEB_LDR_DATA* GetLdrData(PVOID peb);
    
    // 遍历所有模块
    static bool EnumModules(ModuleEnumCallback callback, void* context);
    
    // 通过名称查找模块
    static LDR_DATA_TABLE_ENTRY* FindModuleByName(const wchar_t* moduleName);
    
    // 通过基地址查找模块
    static LDR_DATA_TABLE_ENTRY* FindModuleByBaseAddress(PVOID baseAddress);
    
    // 获取当前模块基地址
    static PVOID GetCurrentModuleBase();
};

}
```

#### 2. ModuleHider 类

负责执行Unlink操作：

```cpp
namespace stealth {

class ModuleHider {
public:
    // 隐藏当前模块
    static bool HideCurrentModule();
    
    // 通过名称隐藏模块
    static bool HideModuleByName(const wchar_t* moduleName);
    
    // 通过基地址隐藏模块
    static bool HideModuleByBaseAddress(PVOID baseAddress);
    
    // 检查模块是否已隐藏
    static bool IsModuleHidden(const wchar_t* moduleName);
    
    // 获取错误信息
    static const wchar_t* GetLastErrorMsg();
};

}
```

## 🔗 集成策略

### 在现有模块中调用

```cpp
// 在 modules/helper/helper.cpp 的DllMain中

#include "../stealth/module_hider.h"

BOOL APIENTRY DllMain(HMODULE hModule, DWORD ul_reason_for_call, LPVOID lpReserved) {
    switch (ul_reason_for_call) {
    case DLL_PROCESS_ATTACH:
        {
            // 原有初始化代码...
            
            // 新增：隐藏模块
            #ifdef ENABLE_STEALTH_MODE
            printf("[Helper] Enabling stealth mode...\n");
            if (stealth::ModuleHider::HideCurrentModule()) {
                printf("[Helper] Successfully hidden from module list\n");
            } else {
                printf("[Helper] Failed to hide: %ls\n", 
                       stealth::ModuleHider::GetLastErrorMsg());
            }
            #endif
            
            break;
        }
    case DLL_PROCESS_DETACH:
        // 清理代码...
        break;
    }
    return TRUE;
}
```

## 🧪 测试方案

### 测试工具：test_enumeration.cpp

```cpp
#include <windows.h>
#include <stdio.h>
#include <tlhelp32.h>
#include <psapi.h>

// 测试函数：检查模块是否可见
bool IsModuleVisible(const wchar_t* moduleName) {
    HANDLE hSnapshot = CreateToolhelp32Snapshot(TH32CS_SNAPMODULE, GetCurrentProcessId());
    if (hSnapshot == INVALID_HANDLE_VALUE) {
        printf("Failed to create snapshot\n");
        return false;
    }
    
    MODULEENTRY32W me32;
    me32.dwSize = sizeof(me32);
    
    if (!Module32FirstW(hSnapshot, &me32)) {
        CloseHandle(hSnapshot);
        return false;
    }
    
    bool found = false;
    do {
        if (_wcsicmp(me32.szModule, moduleName) == 0) {
            found = true;
            printf("Found module: %ls at 0x%p\n", me32.szModule, me32.modBaseAddr);
            break;
        }
    } while (Module32NextW(hSnapshot, &me32));
    
    CloseHandle(hSnapshot);
    return found;
}

int main() {
    const wchar_t* targetModule = L"game-payload.dll";
    
    printf("=== Module Visibility Test ===\n\n");
    
    bool visible = IsModuleVisible(targetModule);
    printf("Result: %s\n", visible ? "VISIBLE" : "HIDDEN");
    
    printf("\nPress any key to exit...\n");
    getchar();
    return 0;
}
```

### 测试步骤

```
1. 编译Payload
   → 确保stealth模块已集成

2. 使用Injector注入到测试进程
   → notepad.exe或自定义测试程序

3. 在测试进程中运行test_enumeration.exe
   → 注入前：模块可见
   → 注入后：模块应该隐藏

4. 使用Process Hacker验证
   → 打开Process Hacker
   → 查看目标进程的模块列表
   → game-payload.dll不应该出现

5. 使用Process Explorer验证
   → 同上，验证多种工具

6. 测试DLL功能
   → 确保隐藏后功能仍然正常
   → Helper/Sync仍然工作

7. 生成Minidump测试
   → 使用MiniDumpWriteDump
   → 分析dump中模块列表
   → 应该不包含game-payload.dll
```

## ⚠️ 注意事项

### 安全性

1. **失败不影响功能**
   ```cpp
   try {
       stealth::ModuleHider::HideCurrentModule();
   } catch (...) {
       // 隐藏失败，但DLL仍然工作
       printf("[Stealth] Hide failed, continuing anyway\n");
   }
   ```

2. **多次调用保护**
   ```cpp
   // 避免重复Unlink导致崩溃
   static bool s_hidden = false;
   if (!s_hidden) {
       HideCurrentModule();
       s_hidden = true;
   }
   ```

3. **仅隐藏自己的模块**
   ```cpp
   // 不要隐藏其他DLL
   // 只调用HideCurrentModule()
   ```

### 调试技巧

1. **详细日志**
   ```cpp
   #define STEALTH_VERBOSE 1
   #ifdef STEALTH_VERBOSE
       printf("[Stealth] Detailed info...\n");
   #endif
   ```

2. **逐步验证**
   ```cpp
   // 验证PEB获取
   PVOID peb = PEBUtils::GetCurrentPEB();
   printf("PEB: 0x%p\n", peb);
   
   // 验证模块查找
   auto entry = PEBUtils::FindModuleByName(L"game-payload.dll");
   printf("Found: 0x%p\n", entry);
   
   // 最后才执行Unlink
   ModuleHider::HideCurrentModule();
   ```

3. **使用调试器**
   - 在WinDbg中设置断点
   - 查看PEB结构: `dt nt!_PEB`
   - 查看LDR: `dt nt!_PEB_LDR_DATA`

### 兼容性

1. **x86 vs x64**
   ```cpp
   #ifdef _M_IX86
       // x86代码
   #elif defined(_M_X64)
       // x64代码（结构体偏移不同）
   #endif
   ```

2. **Windows版本**
   - Windows XP: 可能不支持某些API
   - Windows 7+: 全部功能
   - Windows 10/11: 测试重点

3. **DEP/ASLR**
   - 确保编译时正确设置
   - 测试DEP开启的环境
