### Step 1: 创建目录结构（2分钟）

```bash

```

### Step 2: 创建文件（5分钟）

依次创建以下4个文件，完整代码见 [04-代码模板.md](./04-代码模板.md)：

1. `peb_utils.h` - PEB工具头文件
2. `peb_utils.cpp` - PEB工具实现
3. `module_hider.h` - 模块隐藏接口
4. `module_hider.cpp` - Unlink实现

### Step 3: 更新Payload.vcxproj（3分钟）

在Payload项目中添加stealth模块：

```xml
<!-- 在Payload.vcxproj的<ItemGroup>中添加 -->
<ClCompile Include="modules\stealth\peb_utils.cpp" />
<ClCompile Include="modules\stealth\module_hider.cpp" />
<ClInclude Include="modules\stealth\peb_utils.h" />
<ClInclude Include="modules\stealth\module_hider.h" />
```

### Step 4: 集成到现有模块（5分钟）

在 `modules/helper/helper.cpp` 的 DllMain 中添加：

```cpp
#include "../stealth/module_hider.h"

BOOL APIENTRY DllMain(HMODULE hModule, DWORD ul_reason_for_call, LPVOID lpReserved) {
    switch (ul_reason_for_call) {
    case DLL_PROCESS_ATTACH:
        {
            // 现有代码...
            
            // 新增：隐藏模块
            #ifdef ENABLE_STEALTH_MODE
            printf("[Helper] Hiding module from PEB...\n");
            if (stealth::ModuleHider::HideCurrentModule()) {
                printf("[Helper] SUCCESS: Module is now hidden!\n");
            } else {
                printf("[Helper] FAILED: %ls\n", 
                       stealth::ModuleHider::GetLastErrorMsg());
            }
            #endif
            
            break;
        }
    }
    return TRUE;
}
```

### 方法2: 使用测试工具

创建简单的测试程序：

```cpp
// test_visibility.cpp
#include <windows.h>
#include <stdio.h>
#include <tlhelp32.h>

int main() {
    HANDLE hSnapshot = CreateToolhelp32Snapshot(
        TH32CS_SNAPMODULE, 
        GetCurrentProcessId()
    );
    
    MODULEENTRY32W me32;
    me32.dwSize = sizeof(me32);
    
    if (Module32FirstW(hSnapshot, &me32)) {
        do {
            if (wcsstr(me32.szModule, L"game-payload")) {
                printf("FOUND: %ls at 0x%p\n", 
                       me32.szModule, 
                       me32.modBaseAddr);
                CloseHandle(hSnapshot);
                return 1;  // 可见
            }
        } while (Module32NextW(hSnapshot, &me32));
    }
    
    printf("NOT FOUND: game-payload.dll is hidden!\n");
    CloseHandle(hSnapshot);
    return 0;  // 隐藏
}
```

编译运行：
```bash

test_visibility.exe
```

## 🐛 常见问题排查

### 问题1: 编译错误

```
错误: 'GetCurrentPEB': undeclared identifier
解决: 确保包含了 <intrin.h> 和 <winternl.h>
```

### 问题2: 链接错误

```
错误: unresolved external symbol
解决: 将所有.cpp文件添加到vcxproj的<ClCompile>中
```

### 问题3: 运行时崩溃

```
原因: 可能是结构体对齐问题
解决: 确保使用正确的结构体定义
```

### 问题4: Unlink失败

```
原因: 模块已经被Unlink或链表损坏
解决: 添加ValidateLinks检查，避免重复Unlink
```

### 问题5: 模块仍然可见

```
排查步骤:
1. 检查是否调用HideCurrentModule()
2. 检查返回值和错误信息
3. 使用调试器验证PEB结构
4. 确认在正确的时机调用（DLL_PROCESS_ATTACH）
```
